
# Reading Health Report CSV files into a database

# Quick start

In addition to this code, you'll need :

 1. The .env file (so the Python can access the database)
 2. A fairly complete set of csv files as generated by the health report software
    (so one can populate the database)

Given that, the setup procedure is :

 1. In the $HOME directory, clone this repo and cd into the vsoHealthDB/ directory.
 2. Set up the uv python virtual environment by running the *setup_uv.sh* script
    (it is assumed that uv is installed and that a modern python version has been pinned)
 3. Install the git pre commit hook by running the *install_hook.sh* script
 4. Put the .env file into the $HOME/vsoHealthDB/ directory
 5. Put the csvFiles into the directory like $HOME/vsoHealthDB/csvFiles/
 6. Populate the database by running the db_loader.sh script with the
    CSV file directory as an argument, like so : ```./db_loader.sh csvFiles```
 7. Having populated the sqlite database (the file report.db should exist)
    add a cron job that starts the uvicorn server, like so :

```
* * * * * $HOME/vsoHealthDB/cronRunner.sh &> $HOME/vsoHealthDB/cronRunner.log
```

The server can also be started directly with the *startServer.sh* script.
It can also be killed with the *kill.sh* script. Once the server is running,
the web page (which has links to the FastAPI generated documentation) will
be served out on port 26996 from the host in question.
